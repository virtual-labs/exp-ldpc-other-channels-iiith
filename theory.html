Message Passing Decoding of LDPC Codes on Other Channels

We now describe the general message passing algorithm for achieving Bit-wise Maximum Aposterior Probability (MAP) Decoding of linear codes on general channels. We focus on Additive White Gaussian Noise (AWGN) channels, but the principles can be adapted to other channel models.

Bit-wise MAP decoding principle for linear codes

Let x=(x1,…,xn)∈C\bm{x}=(x_1,\ldots,x_n)\in {\cal C}x=(x1​,…,xn​)∈C be the codeword transmitted from the linear code C\cal CC of dimension kkk. Let y=(y1,…,yn)\bm{y}=(y_1,\ldots,y_n)y=(y1​,…,yn​) be the received vector from the channel. The bit-wise MAP estimate of the bit xix_ixi​ is based on the value of the log-likelihood ratio (LLR) of xix_ixi​ given y\bm{y}y, denoted by L(xi∣y)L(x_i|\bm{y})L(xi​∣y), and defined as follows.

L(xi)≜log⁡(p(xi=0∣y)p(xi=1∣y)).(1)L(x_i)\triangleq \log\left(\frac{p(x_i=0|\bm{y})}{p(x_i=1|\bm{y})}\right). \tag{1}L(xi​)≜log(p(xi​=1∣y)p(xi​=0∣y)​).(1)

In the case of AWGN channels, we assume without loss of generality that xi∈{+1,−1}x_i\in\{+1,-1\}xi​∈{+1,−1}, under a bipolar signalling scheme. Thus, the LLR L(xi)L(x_i)L(xi​) is then defined as L(xi)≜log⁡(p(xi=+1∣y)p(xi=−1∣y)).(2)L(x_i)\triangleq \log\left(\frac{p(x_i=+1|\bm{y})}{p(x_i=-1|\bm{y})}\right). \tag{2}L(xi​)≜log(p(xi​=−1∣y)p(xi​=+1∣y)​).(2)

The bitwise MAP estimate for xix_ixi​ is then denoted by x^i\hat{x}_ix^i​ and calculated as follows.

x^i={0,if L(xi)>01,if L(xi)<0.(3) \begin{align*} \hat{x}_i=\begin{cases}0,& \text{if}~L(x_i)>0\\ 1, & \text{if}~L(x_i)<0. \end{cases} \end{align*} \tag{3} x^i​={0,1,​if L(xi​)>0if L(xi​)<0.​​(3)

Note that we assume ties are broken arbitrarily, i.e., the decoder choose the estimate x^i\hat{x}_ix^i​ randomly if L(xi∣y)=0L(x_i|\bm{y})=0L(xi​∣y)=0.

As before, we assume that the Tanner graph of the code C\cal CC is cycle-free, i.e., the Tanner graph has no sequence of edges between adjacent vertices which start and end at the same vertex. Thus, the Tanner graph is a tree. With this assumption in mind, the belief propagation approach is described on the Tanner graph of the code, which leads to an efficient computation of the Bit-wise MAP of each bit in the codeword transmitted.

Message Passing for LDPC Codes on AWGN Channels

The Min-Sum algorithm is an iterative procedure where messages, representing LLRs, are passed between variable nodes and check nodes on the Tanner graph.

The Min-Sum Decoding Algorithm

The algorithm consists of initialization followed by a series of iterations until a termination condition is met.

Step 1: Initialization

  1. Channel LLRs: For each received value yiy_iyi​ from the AWGN channel with noise variance σ2=N0/2\sigma^2 = N_0/2σ2=N0​/2 and assuming BPSK modulation (0↦+1,1↦−10 \mapsto +1, 1 \mapsto -10↦+1,1↦−1), the initial channel LLR is calculated as: Lch(xi)=2yiσ2(4)L_{ch}(x_i) = \frac{2y_i}{\sigma^2} \tag{4}Lch​(xi​)=σ22yi​​(4) This value represents the initial belief about bit xix_ixi​ from the channel observation alone.
  2. Message Initialization: Messages from check nodes to variable nodes are set to zero before the first iteration. Let Lj→i(k)L_{j \to i}^{(k)}Lj→i(k)​ be the message from check node zjz_jzj​ to variable node xix_ixi​ in iteration kkk. Then, Lj→i(0)=0L_{j \to i}^{(0)} = 0Lj→i(0)​=0 for all edges (xi,zj)(x_i, z_j)(xi​,zj​).

Step 2: Iterative Processing (for iteration k=1,2,…,kmaxk=1, 2, \dots, k_{max}k=1,2,…,kmax​)

A. Variable Node (VN) to Check Node (CN) Update: Each variable node xix_ixi​ sends a message Li→j(k)L_{i \to j}^{(k)}Li→j(k)​ to each adjacent check node zjz_jzj​. This message is the sum of the channel LLR and all incoming messages from other check nodes in the previous iteration. Li→j(k)=Lch(xi)+∑j′∈N(xi)∖{j}Lj′→i(k−1)(5)L_{i \to j}^{(k)} = L_{ch}(x_i) + \sum_{j' \in N(x_i) \setminus \{j\}} L_{j' \to i}^{(k-1)} \tag{5}Li→j(k)​=Lch​(xi​)+j′∈N(xi​)∖{j}∑​Lj′→i(k−1)​(5) where N(xi)N(x_i)N(xi​) is the set of check nodes connected to xix_ixi​.

B. Check Node (CN) to Variable Node (VN) Update: Each check node zjz_jzj​ sends a message Lj→i(k)L_{j \to i}^{(k)}Lj→i(k)​ to each adjacent variable node xix_ixi​. The Min-Sum approximation provides a simplified rule for this update: Lj→i(k)=(∏i′∈N(zj)∖{i}sgn⁡(Li′→j(k)))×min⁡i′∈N(zj)∖{i}∣Li′→j(k)∣(6)L_{j \to i}^{(k)} = \left(\prod_{i' \in N(z_j) \setminus \{i\}} \operatorname{sgn}\left(L_{i' \to j}^{(k)}\right)\right) \times \min_{i' \in N(z_j) \setminus \{i\}} \left|L_{i' \to j}^{(k)}\right| \tag{6}Lj→i(k)​=​i′∈N(zj​)∖{i}∏​sgn(Li′→j(k)​)​×i′∈N(zj​)∖{i}min​​Li′→j(k)​​(6) where N(zj)N(z_j)N(zj​) is the set of variable nodes connected to zjz_jzj​. The message's sign is determined by the product of the signs of incoming messages (to satisfy the parity check), and its magnitude is the minimum of the incoming magnitudes.

Step 3: LLR Aggregation and Decision After the check node update, each variable node aggregates all its information to compute a total LLR and make a hard decision.

  1. Total LLR: Ltotal(k)(xi)=Lch(xi)+∑j∈N(xi)Lj→i(k)L_{total}^{(k)}(x_i) = L_{ch}(x_i) + \sum_{j \in N(x_i)} L_{j \to i}^{(k)}Ltotal(k)​(xi​)=Lch​(xi​)+∑j∈N(xi​)​Lj→i(k)​.
  2. Hard Decision: A temporary decoded vector x^(k)\hat{\bm{x}}^{(k)}x^(k) is formed using the rule: x^i=0\hat{x}_i = 0x^i​=0 if Ltotal(k)(xi)≥0L_{total}^{(k)}(x_i) \ge 0Ltotal(k)​(xi​)≥0, and x^i=1\hat{x}_i = 1x^i​=1 otherwise.

Step 4: Termination The iterations stop if either of these conditions is met:

  1. Valid Codeword: The hard decision vector x^(k)\hat{\bm{x}}^{(k)}x^(k) satisfies all parity checks (i.e., H(x^(k))T=0H(\hat{\bm{x}}^{(k)})^T = \bm{0}H(x^(k))T=0). The algorithm terminates successfully.
  2. Maximum Iterations: The algorithm reaches a predefined maximum number of iterations, kmaxk_{max}kmax​. If the result is not a valid codeword, a decoding failure is declared.
